# Knowledge Base

This directory stores persistent aesthetic profiles generated by the `aesthetic-research` and `image-analysis` skills.

## Structure

```
skills/aesthetic-research/knowledge/
└── schema.json            # JSON Schema for profile frontmatter (canonical spec)

knowledge/aesthetics/      # at workspace root — runtime write target
├── <slug>.md              # Aesthetic profile (committed, append-only)
└── <slug>/
    ├── images/            # Downloaded reference images (git-ignored, local-only)
    └── generated/         # Generated assets (git-ignored, local-only)
```

## Conventions

- **Slug format**: lowercase-hyphenated (e.g., `dark-academia`, `frutiger-aero`, `neubrutalism`)
- **Append-only**: Profiles are never overwritten. New research runs append a dated `## Research Update YYYY-MM-DD` section at the bottom.
- **Images**: Local-only via `.gitignore`. Store in `knowledge/aesthetics/<slug>/images/` after visual research.
- **Generated assets**: Local-only via `.gitignore`. Stored in `knowledge/aesthetics/<slug>/generated/` by `asset-creation`.
- **New aesthetics**: Profiles for aesthetics not in the `aesthetic-literacy` dictionary include `new_aesthetic: true` in their YAML frontmatter.

## Profile Frontmatter Fields

The canonical schema is defined in [`schema.json`](schema.json) (co-located with the `aesthetic-research` skill). Summary:

```yaml
---
slug: example-aesthetic
label: Example Aesthetic
first_researched: 2026-01-01
last_updated: 2026-01-01
source: <image-source>     # e.g. pinterest, dribbble, behance, mixed
image_count: 17
evidence_level: standard   # standard | limited (< 10 images)
new_aesthetic: false       # true = not in aesthetic-literacy dictionary; candidate for promotion
---
```
